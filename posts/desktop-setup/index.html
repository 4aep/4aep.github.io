  • other:

    why use Emacs

Emacs (run through wsl) is by far the most capabable keyboard centric editor. VS Code is fine tuned for the mouse. I tried to configure and use more more keyboard-centric but it is kinda messy.

workflow / window management

I have setup similar to this post.

  • use floating windows

  • use one virtual desktop

  • 1 key to toggle max/restore window size. F1

  • 1 key to switch

  • mouse hover auto-raise

  • set-up 3 function keys to switch to 3 most used apps (currently one have set this to be chrome will do emacs and probably WSL terminal)

  • fuzzy search to get window

  • set up 1 key to close window. F6

  • mouse hover auto-raise

  • no double click

  • remove desktop icons
